# Phase 7: Retrospective

**Purpose:** Structured reflection on the process to continuously improve the governance framework itself.

---

<!-- depth:shallow -->
## Shallow Depth (Standard Pathway)

### Quick Retro
Answer three questions:

1. **What went well?** — Which parts of the governance process added clear value?
2. **What didn't work?** — Where did the process create friction without value?
3. **One improvement** — What is one concrete change to make next time?

### Artifact
```markdown
## Retrospective: [Task Name]
**Date:** [date]

### Went Well
- [item]

### Didn't Work
- [item]

### Improvement
- [concrete action]
```

### Actions
- If a rule was too strict or too lax, propose an update to the rule file.
- If a phase was unnecessary for this pathway, note it for pathway calibration.

## Deep Depth (Complex Pathway)

*Includes everything from Moderate, plus:*

### Extended Analysis

#### Impact Assessment
- Did the governance process prevent any actual defects? List them.
- Were there any defects that the governance process missed? Analyze why.
- What was the total overhead vs. estimated defect prevention value?

#### Framework Evolution Proposals
Based on this experience, propose concrete changes to:
1. **Rules** — New rules needed? Existing rules to modify? Rules to retire?
2. **Phases** — Phase depth calibration changes? New phase activities?
3. **Pathways** — Should pathway thresholds be adjusted?
4. **Assessment** — Were the complexity factors weighted correctly?

#### Knowledge Base Updates
- New architectural patterns established → Update architecture rules
- New security considerations discovered → Update security rules
- New performance patterns identified → Update performance rules

### Artifact
Comprehensive retrospective with framework evolution proposals. Changes approved here feed back into the governance framework for the next cycle.

### 🛑 FINAL CHECKPOINT
Review retrospective findings. Approve or modify proposed framework changes. Update governance files accordingly.
